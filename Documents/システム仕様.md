# システム仕様

Claude Developer Tool（Electron + React + TypeScript + MUI）のシステム仕様。

## 概要

開発者向けに Claude Desktop と Claude Code（CLI）のユーティリティを提供する GUI ツール。起動時はダッシュボードを表示し、以下の画面で構成される。

| 画面 | ルート | 機能 |
| --- | --- | --- |
| ダッシュボード | `/` | 3 機能をカードで選択する起動画面 |
| Claude Desktop MCP 管理 | `/claude-desktop` | Claude Desktop の MCP サーバーの有効化/無効化/並べ替え/再起動 |
| Claude Code MCP 管理 | `/claude-code` | Claude Code（CLI）のプロファイル MCP の有効化/無効化/並べ替え |
| Claude Code クリーンアップ | `/cleanup` | `~/.claude` 配下の履歴・キャッシュ・一時・ログの削除 |

## ナビゲーション

- React Router（開発時 `BrowserRouter`、本番 `HashRouter`）でルートを切り替える。
- 起動時は `/`（ダッシュボード）を表示し、3 機能をカードで選択して各画面へ遷移する。
- 画面切替の手段は 3 つ:
  1. ダッシュボードのカード
  2. タイトルバーのバーガーメニュー（全画面項目）
  3. タイトルバーの現在画面キャプション（クリックでドロップダウン、全画面を選択可能）
- 現在画面は `location.pathname` で `selected` 表示。タイトルバーのアプリ名横にキャプションを表示し、ダッシュボードアイコンからいつでもダッシュボードへ戻れる。

## ClaudeEnvironment 抽象

すべての Claude Code 系操作は `ClaudeEnvironment`（`{ kind: 'native' | 'wsl'; distro? }`）を対象とする。

- `native` は常に 1 つ存在（ホスト OS）。
- `wsl` 環境は Windows かつ distro 内に `~/.claude.json` が存在する場合のみ列挙する。
- `ClaudeFs` がこの記述子を具体的なファイル操作（read/write/list/delete/size）に解決し、native の `fs` と WSL のアクセス方式を隠蔽する。各マネージャーは `ClaudeFs` にのみ依存する。

### WSL 検出（`WslDetector`）

- `wsl -l -q` の出力は環境により UTF-16LE（NUL 埋め）または UTF-8。NUL バイト密度から自動判定して復号する。
- distro の Linux ホームは `wsl -d <distro> -- bash -lc 'echo -n $HOME'` で解決する（`/home/<windowsuser>` を仮定しない）。
- `~/.claude.json` の存在で Claude Code 導入済み distro を判定する。
- 検出結果はプロセス起動中キャッシュする。

### WSL ファイルアクセス（`ClaudeFs`）

- 優先: `\\wsl.localhost\<distro>\<home>` → `\\wsl$\<distro>\<home>` の UNC パスを通常の `fs` で操作。
- フォールバック: UNC 不可時は `wsl -d <distro> -- bash -lc '...'` のコマンドモード（`cat`/`tee`/`rm -rf`/`du -sb`/`find`）。書き込みは base64 経由でエスケープ問題を回避する。

## Claude Code MCP 管理（`ClaudeCodeManager`）

- グローバル MCP は `~/.claude.json` のトップレベル `mcpServers`。
- 無効化エントリは `~/.claude-disabled-mcp.json`（サイドカー、`{ mcpServers }`）へ退避。空になると削除する。
- **外科的編集ポリシー**: `~/.claude.json` はフルオブジェクトを読み込み、`mcpServers` キーのみを変更して `JSON.stringify(full, null, 2)`（2 スペースインデント）で書き戻す。`projects` などその他のキーと順序は保持される。
- エントリはオブジェクトとしてそのまま移動し、`type`（例 `"stdio"`）等のフィールドを落とさない。
- 並べ替えは指定順に再構築し、未指定キーは末尾に追加する。
- プロジェクト単位 MCP（`projects[path].mcpServers`）は対象外。
- 再起動概念はない（CLI のため）。

## Claude Code クリーンアップ（`ClaudeCleanupManager`）

- 対象は履歴／キャッシュ／一時／ログのディレクトリのみ（`CLEANUP_CANDIDATES`）。
- 各候補の存在とサイズ（再帰バイト数）をスキャンして返す。`projects` はサブディレクトリ（プロジェクト別）の名前とサイズを `children` として返す。
- 削除:
  - `projects` 以外、および `projects` 全体指定は **ディレクトリごと削除**（Claude Code が必要時に自動再作成する）。
  - `projects` の個別プロジェクト削除は、`projects` 全体が選択されていない場合のみ、対象サブディレクトリを削除する。
  - 削除後に再スキャン結果を返す。一部失敗時は失敗キーを含む例外を投げる。
- **対象外（カタログに含めない）**: インストール資産 `plugins`/`skills`、稼働中ランタイム状態 `daemon`/`ide`、設定 `jobs`/`teams`。

### クリーンアップ候補（`CLEANUP_CANDIDATES`）

デフォルトはすべてチェック OFF（ユーザーが明示的に選択する）。

| キー | 内容 |
| --- | --- |
| projects | プロジェクト別セッション履歴・メモリー（最大容量。個別選択可） |
| file-history | ファイル編集履歴スナップショット |
| shell-snapshots | シェル環境スナップショット |
| cache | キャッシュ（changelog 等） |
| debug | デバッグログ |
| sessions | セッション状態 |
| session-env | セッション別環境 |
| tasks | タスク状態・一時ロック |
| backups | 旧 `~/.claude.json` バックアップ（セーフティネット） |

## IPC チャンネル

### Claude Desktop（既存）

- `claude-desktop:get-info` / `:get-mcp-servers` / `:enable-mcp-server` / `:disable-mcp-server` / `:reorder-mcp-servers` / `:reorder-disabled-mcp-servers` / `:restart`

### Claude Code MCP

- `claude-code:get-environments`
- `claude-code:get-mcp-servers`
- `claude-code:enable-mcp-server`
- `claude-code:disable-mcp-server`
- `claude-code:reorder-mcp-servers`
- `claude-code:reorder-disabled-mcp-servers`

### Claude Code クリーンアップ

- `claude-cleanup:get-environments`
- `claude-cleanup:scan`
- `claude-cleanup:delete`

preload はサンドボックス下のため共有定数を import できず、チャンネル文字列をインラインで保持する（`shared/constants.ts` と一致させる）。

## セキュリティ・安全策

- **パストラバーサル対策**: 削除対象のディレクトリ名／プロジェクト名は既知カタログ・実在サブディレクトリと照合し、`/`・`\`・`..` を含む名前は拒否する。
- **WSL は Windows 限定**: すべての WSL コードパスは `process.platform === 'win32'` でガードする。
- **巨大ファイル整合性**: `~/.claude.json` は全体再構築せず `mcpServers` のみ変更する。
- **使用中ディレクトリ**: 削除はディレクトリ単位で try/catch し、ロック等による部分失敗を許容して他を継続、結果を通知する。
- **symlink ループ**: 再帰サイズ計算で symlink をスキップする。
