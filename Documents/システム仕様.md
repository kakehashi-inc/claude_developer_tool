# システム仕様

Claude Developer Tool（Electron + React + TypeScript + MUI）のシステム仕様。

## 概要

開発者向けに Claude Desktop と Claude Code（CLI）のユーティリティを提供する GUI ツール。起動時はダッシュボードを表示し、以下の画面で構成される。

| 画面 | ルート | 機能 |
| --- | --- | --- |
| ダッシュボード | `/` | 3 機能をカードで選択する起動画面 |
| Claude Desktop MCP 管理 | `/claude-desktop` | Claude Desktop の MCP サーバーの有効化/無効化/並べ替え/再起動 |
| Claude Code MCP 管理 | `/claude-code` | Claude Code（CLI）のプロファイル MCP の有効化/無効化/並べ替え |
| Claude Code クリーンアップ | `/cleanup` | `~/.claude` 配下の履歴・キャッシュ・一時・ログの削除 |

## ナビゲーション

- React Router（開発時 `BrowserRouter`、本番 `HashRouter`）でルートを切り替える。
- 起動時は `/`（ダッシュボード）を表示し、各機能をカードで選択して各画面へ遷移する。
- 画面の並び順は ダッシュボード → Claude Desktop MCP 管理 → Claude Code MCP 管理 → **Claude Code Agent・Skill管理** → Claude Code クリーンアップ。Agent・Skill管理 はクリーンアップの手前に配置する。
- 画面切替の手段は 3 つ:
  1. ダッシュボードのカード
  2. タイトルバーのバーガーメニュー（全画面項目）
  3. タイトルバーの現在画面キャプション（クリックでドロップダウン、全画面を選択可能）
- 現在画面は `location.pathname` で `selected` 表示。タイトルバーのアプリ名横にキャプションを表示し、ダッシュボードアイコンからいつでもダッシュボードへ戻れる。

### 再取得（各機能画面共通）

- ダッシュボードを除く各機能画面（Claude Desktop MCP 管理 / Claude Code MCP 管理 / Agent・Skill管理 / クリーンアップ）は、大見出しの右に「再取得」ボタン（`common.refresh`、`Refresh` アイコン、`variant='outlined'`）を表示する。
- 押下すると `loading` を `true` に戻して当該画面のデータ取得処理（`getEnvironments` / `getInfo` 等）を最初から再実行し、画面を丸ごと再読み込みする。読み込み中は各画面共通のローディング表示（`common.loading`）に切り替わる。
- Claude Desktop MCP 管理では「再起動」ボタンの左に並べて配置する。再起動アイコンは再取得アイコンと区別するため `RestartAlt` を用いる。

## ClaudeEnvironment 抽象

すべての Claude Code 系操作は `ClaudeEnvironment`（`{ kind: 'native' | 'wsl'; distro? }`）を対象とする。

- `native` は常に 1 つ存在（ホスト OS）。
- `wsl` 環境は Windows かつ distro 内に `~/.claude.json` が存在する場合のみ列挙する。
- `ClaudeFs` がこの記述子を具体的なファイル操作（read/write/list/delete/size）に解決し、native の `fs` と WSL のアクセス方式を隠蔽する。各マネージャーは `ClaudeFs` にのみ依存する。

### WSL 検出（`WslDetector`）

- `wsl -l -q` の出力は環境により UTF-16LE（NUL 埋め）または UTF-8。NUL バイト密度から自動判定して復号する。
- distro の Linux ホームは `wsl -d <distro> -- bash -lc 'echo -n $HOME'` で解決する（`/home/<windowsuser>` を仮定しない）。
- `~/.claude.json` の存在で Claude Code 導入済み distro を判定する。
- 検出結果はプロセス起動中キャッシュする。

### WSL ファイルアクセス（`ClaudeFs`）

- 優先: `\\wsl.localhost\<distro>\<home>` → `\\wsl$\<distro>\<home>` の UNC パスを通常の `fs` で操作。
- フォールバック: UNC 不可時は `wsl -d <distro> -- bash -lc '...'` のコマンドモード（`cat`/`tee`/`rm -rf`/`du -sb`/`find`）。書き込みは base64 経由でエスケープ問題を回避する。

## Claude Code MCP 管理（`ClaudeCodeManager`）

- グローバル MCP は `~/.claude.json` のトップレベル `mcpServers`。
- 無効化エントリは `~/.claude-disabled-mcp.json`（サイドカー、`{ mcpServers }`）へ退避。空になると削除する。
- **外科的編集ポリシー**: `~/.claude.json` はフルオブジェクトを読み込み、`mcpServers` キーのみを変更して `JSON.stringify(full, null, 2)`（2 スペースインデント）で書き戻す。`projects` などその他のキーと順序は保持される。
- エントリはオブジェクトとしてそのまま移動し、`type`（例 `"stdio"`）等のフィールドを落とさない。
- 並べ替えは指定順に再構築し、未指定キーは末尾に追加する。
- プロジェクト単位 MCP（`projects[path].mcpServers`）は対象外。
- 再起動概念はない（CLI のため）。

## Claude Code Agent・Skill管理（`AssetManager`）

`~/.claude/agents` および `~/.claude/skills` 配下のサブディレクトリ（= 各エージェント / 各スキル）を ZIP で入出力する。クリーンアップ画面の手前に配置する。

- **対象環境**: native ＋ Claude 入り WSL distro（`WslDetector.getClaudeDistros()`、`~/.claude.json` で判定）。agents/skills が未作成でもアップロード先として列挙する。
- **一覧の単位（種別で異なる）**:
  - **skills**: `skills/` 直下の各サブディレクトリを 1 件とする。frontmatter は `<skill>/SKILL.md` から読む。ファイル数のみ `dirStats`（再帰）で取得する（サイズは表示しないため取得・保持しない）。
  - **agents**: `agents/` 直下の `.md` ファイル、およびサブディレクトリ配下の `.md` ファイル（再帰、最大深さ 5）を 1 件ずつ展開する。ディレクトリ自体はエージェントではなく、その配下の `.md` を対象とする。frontmatter は各 `.md` から読む。1 ファイル固定のためサイズ・ファイル数は取得しない（表示もしない）。
- **取得方針**: UI で表示しない値（agents のサイズ・ファイル数、skills のサイズ）は **そもそも取得・計算しない**（無駄な stat / 走査を避ける）。`AssetEntry.fileCount` は skills のみ設定する任意項目。
- **frontmatter 読み取り**: `src/main/utils/frontmatter.ts`（読み取り専用・依存なし）。先頭の `---` で囲まれたヘッダー部から、トップレベルの `key: value` スカラーと生テキスト（`raw`）を抽出する。各エントリは `frontmatter`（key→value）と `frontmatterRaw`（生テキスト）を持つ。
- **一覧（`list(env, kind)`）**: 上記の単位で列挙し、各エントリに `relPath`（asset 親からの相対パス。DL/UL の単位）・`isFile`・（skills のみ）`fileCount`・frontmatter を付与して名前順に返す。
- **ダウンロード（`download(env, kind, relPaths, window)`）**: 保存先を `dialog.showSaveDialog`（既定名 `<kind>[-<distro>].zip`）で指定し、`archiver`（zip, level 9）で選択した各エントリを ZIP に追加する。**複数選択でも 1 つの ZIP** にまとめる。
  - skills: `<name>/` としてディレクトリごと追加。
  - agents: **agents ディレクトリからの相対パスを維持**して `.md` ファイルを追加（サブディレクトリ構造を保持）。
- **アップロード前検査（`inspectUpload(env, kind, window)`）**: `dialog.showOpenDialog`（ZIP 単一選択）で ZIP を選び、`adm-zip` で内容を読む。衝突判定は種別で異なる。
  - skills: ZIP のトップレベルディレクトリ名 vs 既存サブディレクトリ。
  - agents: ZIP の各ファイルの相対パス vs 既存ファイル（同一パスがあれば衝突）。
  選択 ZIP の実パス（`zipPath`）と衝突一覧（`conflicts`）を返す。
- **アップロード（`upload(env, kind, zipPath, overwrite)`）**: `overwrite=true` の場合、置き換え単位ごとに既存を削除してから `adm-zip` の `extractAllTo(parent, true)` で対象フォルダへ展開する。展開した件数を `importedCount` として返す。
  - **skills（ディレクトリ単位）**: ZIP に含まれる各トップレベルディレクトリについて、既存があれば **ディレクトリごと削除**してから展開する。これにより対象スキルは古いファイルが残らないクリーンな置換になる（`extractAllTo` はファイル単位上書きのため、削除を前提とするディレクトリ置換にはこの明示削除が必要）。
  - **agents（ファイル単位）**: ZIP の各ファイルについて、同一相対パスの既存ファイルを削除してから展開する（ファイル単位の上書き）。
- **削除（`deleteSelected(env, kind, relPaths)`）**: 選択したエントリを削除する。skills はディレクトリごとベストエフォート削除（`removeDirBestEffort`、使用中で消せない分は `skipped` に報告）、agents は `.md` ファイルを削除する。例外は投げず、削除できた件数（`deletedCount`）と消せなかった対象（`skipped`）を返す。
- **UI**: 環境ごとにセクションを並べ（native 群 → Divider → WSL 群）、各セクション内で「エージェント」「スキル」を **タブで分離**。各タブは Paper + Table（`table-layout: fixed` で列幅を安定化）。frontmatter を固定列で展開する（**agents**: name / tools / model / description、**skills**: name / description）。説明列は残り幅を使う伸縮列。各セルは最大 2 行まで表示し超過分は省略（…）＋ Tooltip で全文表示。name セルには agents のサブディレクトリ相対パスを控えめ（`text.disabled`）に併記する。ファイル数列は **skills のみ**表示（agents は非表示）、サイズ列は廃止。**「参照」ボタン**でそのエントリの生 frontmatter 全体をダイアログ（等幅・原文忠実）で表示する。操作ボタンは ダウンロード / アップロード / 削除。同名衝突時は確認ダイアログを表示し、同意で上書き（skills=丸ごと置換 / agents=ファイル置換）する。削除は確認ダイアログのうえ実行し、使用中などで一部消せない場合は警告通知する。
- **実パス前提**: ZIP 圧縮・解凍は実 OS パス（native 絶対パス / WSL UNC パス）に対する通常の `fs` で行う。`ClaudeFs.resolveRealPath` が `null`（WSL コマンドモードで UNC 到達不可）の環境は `available:false` とし、UI で操作不可表示にする。
- **安全策**: サブディレクトリ名・相対パスは `/`・`\`・`..`・絶対パス・ドライブ指定を含むものを拒否。展開する ZIP は全エントリ名を検査し、不正なら展開しない（パストラバーサル対策）。

### md 単体アップロード（ZIP に追加）

ZIP に加えて、`.md` ファイル単体をアップロードできる。`inspectUpload` のファイルダイアログは `zip` / `md` を選択でき、選択ファイルの拡張子で取り込み方を分岐する（`AssetOpResult.uploadKind` = `'zip' | 'md'`）。md は `uploadMd(env, kind, mdPath, overwrite)` で取り込む。

- **skills**: ファイル名が（大小無視で）`SKILL.md` の場合は frontmatter の `name` をディレクトリ名にし、`<name>/SKILL.md`（大文字固定）として配置する。`name` が取得できない場合はエラー（`md-no-name`）。`SKILL.md` 以外のファイル名は、拡張子を除き小文字化したものをディレクトリ名にし `<dir>/SKILL.md` として配置する。`overwrite=true` のときは既存ディレクトリを丸ごと削除してから書き込む。衝突判定は `<dir>` の存在。
- **agents**: 元ファイル名のまま `agents/<ファイル名>.md` として配置する。`overwrite=true` のときは既存ファイルを削除してから書き込む。衝突判定は同名ファイルの存在。

### 公式スキルインポート（`anthropics/skills`）

スキルタブのボタン行右端に「公式スキルインポート」ボタンを配置する（`kind === 'skills'` のときのみ表示）。git がインストールされている場合のみ活性（`isGitAvailable()` で判定。`GitRunner` が `git --version` の成否をキャッシュ）。非活性時はツールチップで理由を表示する。

- **リポジトリ管理（`GitRunner`）**: clone 先は `app.getPath('userData')/repos/anthropics-skills`。未存在なら `git clone --depth 1 --branch main <url> <dest>`、既存（`.git` あり）なら `git fetch --depth 1 origin main` → `git reset --hard origin/main` で確実に最新化する。
- **一覧（`listOfficialSkills()`）**: リポジトリを clone/更新後、リポジトリ内 `skills/` 配下を走査し、`SKILL.md`（大小無視）を持つディレクトリを 1 件とする。公式リポは一部スキルが 1 階層ネストするため深さ 2 まで探索する。各件は既存スキル一覧と同形の `AssetEntry`（`relPath` は `skills/` からの相対、`fileCount` は再帰集計、frontmatter を付与）として `AssetOpResult.entries` で返す。
- **インポート（`importOfficialSkills(env, relPaths)`）**: 選択スキルを一時 ZIP（`os.tmpdir()`）に固め、既存アップロードと同じ展開ロジック（`extractZipTo` ＋ skills のディレクトリ置換）で対象環境の `~/.claude/skills` へ展開する。これにより native / WSL（UNC）両方で同じ実績ある経路を通す。WSL の clone/更新は常に **Windows(native) 側 repos** で行い、選択スキルのみを WSL へ運ぶ。**公式スキル同士の置換のため上書き確認は行わず**、同名は削除してから展開する。`resolveRealPath` が `null`（UNC 不可）の場合は `unavailable`。
- **UI**: ボタン押下で公式スキル一覧を取得し、ダイアログ内に **本体一覧と共通の `AssetEntriesTable`**（同レイアウト）で表示する。複数チェック → 「インポート」で取り込み、成功時にダイアログを閉じて一覧を再取得する。

## Claude Code クリーンアップ（`ClaudeCleanupManager`）

- 対象は履歴／キャッシュ／一時／ログのディレクトリのみ（`CLEANUP_CANDIDATES`）。
- 各候補の存在・サイズ（再帰バイト数）・ファイル数をスキャンして返す（`ClaudeFs.dirStats` / `recursiveDirStats`）。`projects` はサブディレクトリ（プロジェクト別）の名前・サイズ・ファイル数を `children` として返す。UI はサイズとファイル数を並べて表示する。
- 削除:
  - `projects` 以外、および `projects` 全体指定は **ディレクトリごと削除**（Claude Code が必要時に自動再作成する）。
  - `projects` の個別プロジェクト削除は、`projects` 全体が選択されていない場合のみ、対象サブディレクトリを削除する。
  - 削除後に再スキャン結果を返す。一部失敗時は失敗キーを含む例外を投げる。
- **対象外（カタログに含めない）**: インストール資産 `plugins`/`skills`、稼働中ランタイム状態 `daemon`/`ide`、設定 `jobs`/`teams`。

### クリーンアップ候補（`CLEANUP_CANDIDATES`）

デフォルトはすべてチェック OFF（ユーザーが明示的に選択する）。

| キー | 内容 |
| --- | --- |
| projects | プロジェクト別セッション履歴・メモリー（最大容量。個別選択可） |
| file-history | ファイル編集履歴スナップショット |
| shell-snapshots | シェル環境スナップショット |
| cache | キャッシュ（changelog 等） |
| debug | デバッグログ |
| sessions | セッション状態 |
| session-env | セッション別環境 |
| tasks | タスク状態・一時ロック |
| backups | 旧 `~/.claude.json` バックアップ（セーフティネット） |

## UI 構成（プラットフォーム単位）

クリーンアップ画面は **プラットフォーム単位**（Windows / macOS / Linux / 各 WSL distro）にセクション（`PlatformCleanupSection`）を並べる。1 つのセクション内に次を縦に並べ、末尾に **1 つの「選択済みを削除」ボタン**を置く（Claude Code の選択とその他ツールの選択をまとめて 1 ボタンで処理）。

1. Claude Code のディレクトリリスト（`~/.claude`、見出しなし）
2. 「その他のツール」リスト（Serena 等、こちらだけサブ見出しを付ける）

表示する環境一覧は `getEnvironments()`（`~/.claude` を持つ環境）と `getOtherEnvironments()`（`~/.serena` を持つ環境）の **和集合**（`envId` で重複排除）。確認ダイアログは件数のみ表示する。

## その他のツールのクリーンアップ（`ClaudeCleanupManager` 拡張）

Claude Code 自体（`~/.claude`）ではない、併用ツール（まず Serena）のクリーンアップを別枠で扱う。各プラットフォームセクション内の「その他のツール」リストとして表示する。

- **汎用項目 registry**（`OTHER_CLEANUP_ITEMS`）: 各項目が「自分の掃除方法」を宣言で内包する。新しい外部ツール項目は registry に定義を 1 つ追加するだけで UI まで反映される。
  - フィールド: `key`, `action`, `targetPath`(HOME 相対), `yamlKey`(任意), `metricKind`, `requiresPath`, `defaultChecked`, `group`。
- **action 種別**（拡張可能）:
  - `dir-delete`: 対象ディレクトリを丸ごと削除。メトリックは size（＋ファイル数）。
  - `yaml-list-clear`: YAML ファイルの対象キー直下のブロックシーケンス（`- ...`）のみをクリア。メトリックは件数（count）。
- **環境検出**: `getOtherEnvironments()` は native ＋（WSL で `~/.serena` を持つ distro）を返す。`~/.serena` 検出は `~/.claude` と独立（`WslDetector.getSerenaDistros()`）。
- **表示条件**: 各項目は `requiresPath` が存在する場合のみ表示。どの環境にも項目が無ければセクションごと非表示。
- **書式保持 YAML 編集**: `src/main/utils/yamlPreserve.ts`（汎用部品）。`clearYamlList` は対象キー行をアンカーに直後のブロックシーケンス項目行のみを行単位で削除し、コメント・他キー・整形・改行コード（CRLF/LF）を完全保持する。**パーサーで再シリアライズしない**（解析の `countYamlListEntries` も同方針のテキスト実装）。
- **テキスト読み書き**: `ClaudeFs.readText` / `writeText`（JSON 変換せずバイト忠実、WSL コマンドモードは base64 経由）。
- **登録項目**:

| キー | action | 対象 | メトリック |
| --- | --- | --- | --- |
| serena-projects | yaml-list-clear | `~/.serena/serena_config.yml` の `projects:` | 登録件数 |
| serena-logs | dir-delete | `~/.serena/logs` | サイズ＋ファイル数 |

- **安全策**: 削除対象キーは registry（`OTHER_VALID_KEYS`）で検証。registry がホワイトリストのためパストラバーサル不可。

## セキュリティ・安全策

- **パストラバーサル対策**: 削除対象のディレクトリ名／プロジェクト名は既知カタログ・実在サブディレクトリと照合し、`/`・`\`・`..` を含む名前は拒否する。Agent・Skill管理では、アップロードする ZIP の全エントリ名も検査し、`..`・絶対パス・ドライブ指定を含む場合は展開を中止する。
- **WSL は Windows 限定**: すべての WSL コードパスは `process.platform === 'win32'` でガードする。
- **巨大ファイル整合性**: `~/.claude.json` は全体再構築せず `mcpServers` のみ変更する。
- **使用中ディレクトリ**: 削除はディレクトリ単位で try/catch し、ロック等による部分失敗を許容して他を継続、結果を通知する。
- **symlink ループ**: 再帰サイズ計算で symlink をスキップする。
